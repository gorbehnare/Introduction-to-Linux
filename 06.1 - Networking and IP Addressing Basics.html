IPv4 Addressing Basics

  • IPv4 address is a 32-bit binary number. For example, the address above: 192.168.56.1 = 1100 0000 1010 1000 0111 000 0000 00001
  • In order to make this number easier to read for humans, it is broken into 4 fields of 8 bits (octets) that are converted to decimal. You get 4 decimal numbers, which are between 0 (0000 0000) and 255 (1111 1111), separated with dots. The machines only work with the 32-bit binary number
  • From the 32 bits, some bits are the address for the network, and some are dedicated to the hosts on that network.
  • The number of bits dedicated to the network is declared using a subnet mask or prefix.
    • The mask is set to 1's for the network bits and 0's for the host bits. Systems will use the logical AND operation to extract the network address from the 32-bit IP address.
    • In the example above, the subnet mask is 255.255.255.0, meaning that the first 24 bits are the network address, and the last 8 bits are the host addresses. This can also be written as /24, so the IP address and Masks in prefix format is: 192.168.56.1/24.
    • When a router or any system needs to extract the network address from any IP address in the example above, the IP is AND with the subnet mask: 192.168.56.153 AND 255.255.255.0 = 192.168.56.0. Therefore, my network address and prefix for the above example is 192.168.56.0/24
    • Network addresses always end with all host bits set to 0. The network address is reserved, and you cannot assign it to a host (according to RFC standards and conventions).
  • The broadcast address refers to ALL hosts inside a network.
    • By RFC/Convention, the Broadcast address is when all host bits are set to 1s
    • A broadcast address from a network is also reserved and cannot be given to a host.
    • In my example network, the broadcast address is 192.168.56.255

Summary

In the above example screenshots from my Virtualbox settings:

  • My network address is 192.168.56.0
  • My mask is 255.255.255.0 or /24
  • My first host IP address is assigned to the virtual adapter in my host OS with the address 192.168.56.1
  • VirtualBox has assigned the address 192.168.56.2 to its internal DHCP server
  • The DHCP server is given a pool of addresses for it to choose from. It will pick and address between 192.168.56.100 and 192.168.56.254 and will assign to any hosts
  • Each host MUST get a unique IP address within a network (two hosts having the same IP is called an IP conflict or IP collision, and things will not work).
  • My available pool of addresses to assign statically to your hosts, while avoiding any possible conflicts with your DHCP server, is from 192.168.56.3 (since .1 and .2 are already taken) to 192.168.56.99
