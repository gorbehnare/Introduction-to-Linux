09 - File Permissions and Notations

File Permissions and Utilities

Every file in a Linux or Unix filesystem has permissions assigned to it. Generic permissions for files and directories are: Read (r), write (w), and execute (x). You can simply view the permissions set on files and directories using ls -l.

Additionally, each file is assigned to a user and a group as the owner. The 3 sets of permissions correspond to permissions for the user owner, group owner, and others (everyone else).

Here are some utilities that you can use to modify these properties:

  • chown: change the ownership of the file (the user and group that owns a file)
  • chmod: Modify the permissions mode, the rwx for user, group, and others
    • chmod [options] mode[,mode] file1 [file2 ...]
  • chgrp: change group ownership

You must have write (w) permission to be able to issue these commands against a file. You cannot issue any of these commands to modify permissions in a way that your user will not have the write permission after the modification. You may need root access if that is the desired outcome.

The permissions can be represented as symbolic (rwx) or numeric (777). Additionally, users, groups, and others can be targeted using letters u, g, and o. Here are some examples of how this can be used:

  • chmod 777 <filename> will set permissions to -rwxrwxrwx on the file
  • chmod u+x will change -rw-r--r-- to -rwxr--r-- (added x for user)
  • chmod o-w will change -rwxrwxrwx to -rwxrwxr-x (removed the w from other)
  • chmod g+rwx will change -rwx------ to -rwxrwx---

The setuid, setgid, and sticky Bits

There are additional bits of permissions that provide more control over access in a shared computing environment. These bits are displayed as an additional digit in front of the regular numeric permissions. For example, 700 is actually 0700 in the filesystem, but we did not type the leading 0 before. Here is an example of how changing these bits looks:

  • chmod 2755 <directory_name>

In symbolic notation, the letter s is applied to a user or group to set the corresponding setuid and setgid bits. For example, to set the setgid bit, we can do the following:

  • chmod g+s <directory_name>, it turns drwxrwxr-x to drwxrwsr-x (notice the s for group permissions)

Additionally, there is a sticky that is denoted by the letter t under other. Here is an example of where this is applied in your Ubuntu Linux system right now:

~$ ls -ld /tmp
drwxrwxrwt 38 root root 860 Sep 16 15:41 /tmp
~$ ls -ld /tmp
drwxrwxrwt 38 root root 860 Sep 16 15:41 /tmp

Notice the t instead of the x for other in the permissions mode.

Here is a summary of what these extra permissions do:

Permission Numeric Symbolic If On Files If On Directories
SUID 4 u+s Users execute file with permissions of the file owner No meaning
SGID 2 g+s Users execute file with permissions of the group owner File created in directory gets the same group owner
Sticky Bit 1 +t No Meaning Users are prevented from deleting other user’s files

For detailed information about setuid, setgid, and sticky bit, refer to man chmod. For more information about the process that gets applied when these additional bits are set, refer to man setuid and man setgid.

Additional Security Controls

For secure environments or for sensitive or critical systems, there are additional access control options available. These additional controls are not within the scope of this course; however, it may be useful to know that such options are available and pursue them if needed.

  • AppArmor (ships with Ubuntu)
    • See AppArmor.net
    • Runs as an additional Linux Kernel Module (runs with the kernel's permission, and alongside your existing kernel)
    • Uses predefined profiles for applications and executables to implement a MAC (Mandatory Access Control). This will prevent the application from deviating from its predefined behaviour
      • Profiles are located under /etc/apparmor.d/, other settings are located under /etc/apparmor
    • Profiles can be in unconfined (do nothing), complaint (only log complaints for non-compliance), and enforce (block any non-conforming behaviour) modes
      • There are also kill and prompt modes
    • You can use aa-status to see the status of your AppArmor operation modes
  • SELinux (Not available on all distros)
    • see the Wikipedia Article
    • Open-source, and created by the United States of America's National Security Agency (the NSA)
    • Must be pre-compiled into the Linux Kernel, so you must install and boot into the SE-Linux variant of the kernel
    • Has its own user accounts, groups and permissions, separate from the Linux systems
    • Implements MAC (Mandatory Access Control) for users and applications
  • Extended File Attributes, but NOT supported in all Linux Distros
    • see Wikipidia Article and Arch Wiki Article)
    • Can provide ACL (Access Control Lists) on files, and add additional attributes such as predefined checksum, or other information
    • Can be used to preserve attributes of files for other Operating Systems (for example, preserve Windows file attributes, even when the file is saved to a Linux/Unix filesystem)
