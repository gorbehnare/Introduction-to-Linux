Adding storage

Step 1: Add a disk to your system

In VirtualBox, you can add additional virtual disks to your VM. Your VM needs to be powered off. Edit your VM settings and create and add a new virtual disk to your system:

  1. Machine > Settings and select Storage.
  2. Click on your storage controller and use the icon to add a disk
  3. In the new window, create a new disk. I choose to create a 5 GB virtual disk.

    You should be able to start your machine at this point.

Step 2: Find the logical name of the disk

There are many ways you can explore hardware in Linux. You can try the following commands to find the name of the new disk you have added:

sudo lshw -C disk
sudo lshw -C disk

You can also use this command to see what the partition structure on the disk currently looks like (hint: the new disk is not yet partitioned!).

lsblk
lsblk

The lsblk utility also shows all the partitions on all disks, so you make sure to note the name of your disk, so you don’t delete partitions by mistake or create partitions on the wrong disk. Removing the wrong partition or formatting the wrong disk can cause data loss and potentially damage to your Linux system. So please make sure you are working on the correct disk and partition.

In my system, the new disk seems to show up as sdb located at /dev/sdb (this may vary in your system).

Step 5: mounting the storage volume

We need to create a mount point to be able to access our new partition. Mount point is simply an empty directory anywhere in your Linux system that you have access to. You can create mount points depending on your needs and your desire.

For this example, I will create a mount point in my home directory, in this case, /home/ali/backups.

ali@ers20059995:~$ pwd
/home/ali
ali@ers20059995:~$ mkdir backups
ali@ers20059995:~$ pwd
/home/ali
ali@ers20059995:~$ mkdir backups

Once we have our desired location as a mount point, we can manually mount the drive.

ali@ers20059995:~$ sudo mount /dev/sdb1 ./backups
ali@ers20059995:~$ sudo mount /dev/sdb1 ./backups

You can use lsblk to verify your new partition is now mounted to your desired location.

Step 6: Automatic Mounting of the storage volume at boot

According to the man pages, the fstab file “contains descriptive information about the filesystems the system can mount.” Read the man pages for fstab before continuing.

If you read the man pages for fstab, you will find that each line describes a filesystem, and each filesystem has several fields that are separated by spaces or tabs. You can make a note of what each field should contain. Using this information, you can create a line with all the appropriate fields and options that you will need for your new storage volume.

Add an entry to your fstab file using a text editor. In this case, I use nano with -B and w options (look those up to see what they do).
Notice: Make sure NOT to modify other filesystem entries, or you can potentially break your system! If you make this mistake, you will need to recover your system by using a bootable media into Linux, and mounting your system root partition to navigate and edit your /etc/fstab file.

ali@ers20059995:~$ sudo nano -Bw /etc/fstab
ali@ers20059995:~$ sudo nano -Bw /etc/fstab

In my system, the fstab entry will look like this:

/dev/sdb1 /home/ali/backups ext4 defaults 0 2
/dev/sdb1 /home/ali/backups ext4 defaults 0 2

Once the fstab entry is saved, you can either reboot your Linux machine to observe the drive being mounted automatically, or just use the following command:

ali@ers20059995:~$ sudo mount -a
ali@ers20059995:~$ sudo mount -a

Verify that your file system is mounted and is writable by your user. Additionally, you can copy some files into your mount point and use the df command to verify how much storage space you have left on each filesystem.
