Introduction

Not all the software you want to use may be available from a repository for your Linux Distro. You will have to install software for business or personal use from other sources at some point. Many developers and enterprises will not take the time to maintain packages in a repository; instead, they will distribute or sell their software via various other methods. Additionally, not all software is open-source or free.

Installing software from internet sources outside of a repository can be more risky. Firstly, you must trust the developer or proprietor of the software. Secondly, you must make sure the software you download is genuine and has not been tampered with. With proprietary software, you must also study the EULAs (End User Licence Agreements). EULAs are inevitably imposed on you and your systems’ users upon encountering software or services. Additionally, companies change the EULA and their Privacy Policies from time to time, and often the new terms are imposed on existing and new users of the software or service.

At this point in this course, you do have enough knowledge to install any software from any source on your Linux system. If you are installing software from outside of repositories for your specific distro, you are responsible for updating and maintaining the software. It is very important to know how to update or remove the specific application since your package management software is not going to be able to do that for you.

One way to find out what the software does to your system during installation is to examine the installation scripts. By examining the installation scripts, you can not only make sure that the installation will work as intended but also figure out what changes it makes to your system in case you want to remove or uninstall the software manually. Since your package management software may not be involved in this at all, you are tasked with taking care of this on your own.

As a practice, we are going to be installing Pi-hole on a Linux VM while examining the installation process and some aspects of this application itself. The reason we are going through the trouble of installing software from outside of repositories is that many commercial products and non-commercial software are distributed this way.

Obtaining and installing Pi-hole

The software and all information are available from www.pi-hole.net. Pi-hole is not a monolithic application. It combines several other software and utilities. Here is a summary of what software packages are used to create Pi-hole (Source: https://docs.pi-hole.net/main/origins/):

  • dnsmasq - a lightweight DNS and DHCP server
  • curl - A command-line tool for transferring data with URL syntax
  • AdminLTE Dashboard - premium admin control panel based on Bootstrap 3.x
  • civetweb - Embedded C/C++ web server
  • sqlite3 - SQL Database engine
  • Pi-hole scripts and tools, such as web UI configurations and the pihole command (https://docs.pi-hole.net/main/pihole-command/)

As you may have noticed, Pi-hole uses a collection of other software to function. A lot of Linux/Unix software is designed this way. In fact, it is encouraged to use existing utilities and tools to create new applications. You can configure and install every package individually in order to achieve the same results, or even better, customize it to your exact needs. However, someone else has already done a lot of the work for us, so we can simply use their creation instead.

Examine The Pi-hole basic-install.sh script

Notice: You should NEVER execute shell scripts from the internet without scrutinizing or at least inspecting the code. This applies to ANY operating system in ANY environment.

In the main documentation (https://docs.pi-hole.net/main/basic-install/), there are 3 methods of installation, all of which involve downloading and running the basic-install.sh bash script. Before we execute this script on the VM, we must examine this script to find out how it works. We can simply download the script file without executing it.

  1. Create a folder called pi-hole in your home directory
    • mkdir pi-hole && cd pi-hole
  2. Obtain the “basic-install.sh” script so we can examine it:
    • wget -O basic-install.sh https://install.pi-hole.net
  3. Now let’s look inside the script file to see what it does:
    • nano basic-install.sh
    • Alternatively, you can copy the script to your host computer and use your favourite text editor to examine it! It may help the readability of the script.

Execute the installation script

Make sure that the basic-install.sh has execute permissions. Go ahead and use sudo bash basic-install.sh to run the installation script.

During the installation, it will ask you to select the network interface for running the server on. Make sure to choose the interface that is host-only with a static IP and not your NAT interface!

Select the desired settings for your DNS provider and lock lists.

NOTE: Make sure that your static IP address is detected properly. Read the IP address and enter the correct IP address if the script is not detecting the correct IP address (i.e. the static IP that you have set for your second network interface). It is crucial that you are reading and applying the correct settings.

On the last screen, the installation should display your randomly generated admin password for the WebUI. Make sure to copy this or write it down. You can change this via terminal if you forget or miss this step (lookup the man pages for pihole to find out how).

Once the setup is done, you can open a web browser on your host and navigate to the IP address of your VM.
