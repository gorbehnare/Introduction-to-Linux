Assignment 1 - Installing Linux

Configuring your Lab Environment

Downloading The Required Files

  1. Download the ISO image for the latest LTS (Long-Term Support) version of Ubuntu Server.
    • https://ubuntu.com/download/server
  2. Download and install VirtualBox on your system
    • Visit https://www.virtualbox.org/ to download the latest version of VirtualBox
    • You can also download and install the matching VirtualBox Extension Pack
      • This step is optional, but is highly recommended

Configure VirtualBox

  1. (Optional, but recommended) Install VirtualBox Extension Pack
    • Select Extensions, and use the Install button to navigate to where you have downloaded the extension pack file in the previous step
  2. Click on Network
    • Under the Host-only Networks tab, if a network does not exist, use the Create button to create a network
    • At the bottom of the window under the Adapter tab, select Configure Adapter Manually. This should give you an already allocated IPv4 address and network mask.
    • Click on the DHCP Server tab and check the Enable Server checkbox.

Create a New Virtual Machine in VirtualBox

  • Under Home (or use Machine), click on New...
  • Give your VM a name, for example COMP2018-VM1
  • Select the following Settings:
    • Select OS as Linux
    • Select OS Distribution as Ubuntu
    • Set OS Version to Ubuntu (64-bit)

  • Click on > Specify Virtual Hardware to expand, set Base Memory to 2048 MB (2 GB), and specify Number of CPUs as 2
  • Click on > Specify virtual hard disk, and under Create a New Virtual Hard Disk set the Disk size to 16 GB

Leave the rest of the settings as default, for example, the Hard Disk Type and Format should be VDI and should NOT have the Pre-allocated Full Size checked

  • Click on Finish to have your new VM created.
  • Select your newly created VM and click on the Settings icon, and select Network. Configure the following Settings:
    • Adapter 1 attached to NAT Network, and choose your NAT network name from the Name drop-down
    • Enable Adapter 2, attached to Host-only Adapter, and select the name of your host-only network

Install Ubuntu Server

  1. Attach your Ubuntu Server installation ISO image file that you had downloaded earlier to the new VM Optical Drive. In the VM page, click on the Optical Drive and select Choose a Disk File.... Navigate and select your Ubuntu Server ISO file.
  2. Start the VM, and it should boot from the installation media and start the setup.
  3. Follow the prompts through the installation and make sure to configure the following settings:
    1. Set your username to be your first name
    2. Set your hostname to be the first 3 letters of your last name + your student ID (for example, ers20059995)
  4. Once the installation is complete, "remove" the optical media from your VM and start your newly installed Linux server.
  5. Take a screenshot of your Virtual Machine application displaying the login prompt of your VM terminal and save it as ScreenShot1-[studentID].png
  6. Log in to your VM using the username and password you had set earlier during the setup.
  7. Type and execute the following two commands (Tip: Linux is case sensitive!):
echo $USER
hostname
echo $USER
hostname
  1. Take a screenshot of your VM terminal showing the above commands and their output and save as ScreenShot2-[studentID].png
  2. Visit Blackboard for instructions on how to submit your screenshots for your assignment

Backup your VM

  1. Shutdown the Linux VM using the command sudo shutdown -h now, or use sudo poweroff
  2. Exit VirtualBox
  3. Navigate to the directory where the VM is stored. In Windows, the default location is under ...\Documents\Virtual Machines. The default location for VirtualBox can be configured. “Default machine Folder” can be found under File > Preferences and under the General screen
  4. Add your Virtual Machine to a Zip Archive. You can back up your zip to another medium, such as a USB thumb drive or another disk, for safekeeping.
  5. You can repeat this process to have a complete backup of your VM every time you make changes, and you are satisfied with the changes
  6. You can also take Snapshots and clone your VM if you are experimenting. Look at the documentation for your Virtualization software

How to restore from a Zip Archive

If something goes wrong with your VM or you need to restore the VM, or even bring the VM to another computer to run, simply delete the existing directory of your VM files (make sure VMware/VirtualBox is NOT running). Replace the directory from the Zip archive by decompressing the files.
