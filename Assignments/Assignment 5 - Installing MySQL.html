Installing MySQL Server on Ubuntu Server

First, update your repository index to make sure your package manager's database is up to date.

sudo apt update
sudo apt update

And upgrade software on your system:

sudo apt upgrade
sudo apt upgrade

Install MySQL server:

sudo apt install mysql-server
sudo apt install mysql-server

Check to see if the MySQL server service is running:

sudo service mysql status
sudo service mysql status

MySQL server is now running on your system, unless there are some errors and issues. However, at this point, MySQL will not start automatically on reboot. You can set the MySQL service to start on startup by using this command:

sudo systemctl enable mysql
sudo systemctl enable mysql

Additionally, you can control the MySQL service using these commands:

sudo service mysql stop
sudo service mysql start
sudo service mysql restart
sudo service mysql stop
sudo service mysql start
sudo service mysql restart

MySQL server default settings have a root account that does not have a password (insecure and dangerous). However, by default, only the local host can access the MySQL server. The MySQL server configurations are in this directory:

/etc/mysql/mysql.conf.d
/etc/mysql/mysql.conf.d

We will revisit configurations in the directory later. Before editing any configurations at this point (and allowing remote login to the server), it is recommended that you configure some security first. We can access the MySQL database itself to set passwords and policies, and manually edit various configurations in files. Instead of changing all the MySQL settings manually to implement many of the recommended settings, you can use this script, which is included with the MySQL package, to help with implementing many of those best practices automatically:

sudo mysql_secure_installation
sudo mysql_secure_installation

NOTE: Read all the questions in each step carefully. Do not forget your MySQL root password.
NOTE: MySQL usernames and passwords ARE NOT the same as your Linux Username and password. The users are stored in the MySQL Database itself and are independent of Linux UID/Password.

I recommend setting the root password to a secure password and NOT allowing the root to access the server remotely. At this point, you can visit the MySQL server configuration files to allow remote connections.

Before we move on, run this command:

sudo ss -tapn | grep mysql
sudo ss -tapn | grep mysql

This should indicate which port (socket) the MySQL server is listening on and from which IP addresses it accepts traffic. You can also look at the MySQL error logs for troubleshooting if you are having any trouble:

sudo tail /var/log/mysql/error.log
sudo tail /var/log/mysql/error.log

Now let’s finally set up MySQL Server's basic configuration so we can connect to it remotely:

sudo nano /etc/mysql/mysql.conf.d/mysqld.cnf
sudo nano /etc/mysql/mysql.conf.d/mysqld.cnf

NOTE: Since we are editing the configuration files, we will need to restart the MySQL service for the changes to take effect. Do not forget to restart the MySQL service to apply new settings (sudo service mysql restart).

In this configuration file, find the line that says port (which is most likely commented out in favour of defaults). Uncomment the line and set your port number to anything that you prefer.

Then find the line with bind-address. This is the IP address that the MySQL service will accept connections from. By default, it only accepts incoming connections from 127.0.0.1 (local host). You can add other bind-address lines to allow additional IP addresses. In this case, since this is a VM and hopefully not exposed to the internet, I do this to simplify our configuration:

bind-address = 127.0.0.1
bind-address = 0.0.0.0
bind-address = 127.0.0.1
bind-address = 0.0.0.0

Save the file and exit. To apply the settings that we just changed, restart the MySQL service:

sudo service mysql restart
sudo service mysql status
sudo ss -tap | grep mysql
sudo service mysql restart
sudo service mysql status
sudo ss -tap | grep mysql

You can get to the MySQL console from your CLI terminal using the mysql command:

sudo mysql -u root
sudo mysql -u root

NOTE: You may have noticed that when configuring mysql_secure_installation, we disabled root access from anywhere but the local machine. Therefore, at this point, only the Linux root account may be able to log into MySQL as root locally (two root accounts are NOT the same; Linux and MySQL have their own accounts). In case it gives you “access denied” due to having set MySQL root to password authentication, you need to supply the MySQL root password by using the option -p:

sudo mysql -uroot -p
sudo mysql -uroot -p

If everything goes well, you should be getting a mysql> prompt.

Let’s add a MySQL user and database so we can use a client application such as MySQL Workbench to access and use the server. Here are the MySQL commands (replace <value> with your own values):

CREATE USER '<Insert Username>'@'%' IDENTIFIED WITH mysql_native_password BY '<insert password>';
CREATE DATABASE <name a database> /*\!40100 DEFAULT CHARACTER SET utf8 */;
GRANT ALL PRIVILEGES ON <name a database>.* TO '<insert username>'@'%';
FLUSH PRIVILEGES;
CREATE USER '<Insert Username>'@'%' IDENTIFIED WITH mysql_native_password BY '<insert password>';
CREATE DATABASE <name a database> /*\!40100 DEFAULT CHARACTER SET utf8 */;
GRANT ALL PRIVILEGES ON <name a database>.* TO '<insert username>'@'%';
FLUSH PRIVILEGES;
