Transferring Files Through SSH (SFTP)

So far you have connected to your Linux VM via SSH using the OpenSSH-clien bundled with your host system. You should not have had to install anything; instead, just launch the Command Prompt in Windows (CMD.exe) or other terminal software, and use ssh username@IP to connect to your Linux VM.

You can use SSH to transfer files. The most basic method is using the scp utility. On your host, navigate to the directory where your files are to be located simply using the cd utility, and use scp to send a file to the remote host. Here is an example syntax:

# this will transmit the "filename" in the current directory to the remote system
scp filename username@IP:/full/path/to/destination

# This will copy a file from the remote system to the local system
scp username@IP:/full/path/to/destination/file localfile

Specifying DNS for your DHCP-enabled NIC

As you may be aware that a DHCP server can provide many settings to clients. The minimum requirements for a client to be able to connect to a network are an IP, a Subnet Mask, and a Gateway (if connecting to an outside network or Internet). However, DHCP can also deliver other settings such as DNS, Domain Names, NTP addresses and many other configurations as needed. Please note that the clients will pick and choose what settings to use, and the DHCP server cannot enforce the clients to configure all settings that are provided.

In this exercise, you will specify custom DNS servers to be used for your first NIC (the iterface connected to NAT or NAT-Network). This adapter's IP, subnet, and gateway are still assigned through DHCP. We will use Netplan in order to make these changes permanent. Please refer to documentation and tutorials on netplan.io to learn how to specify DNS servers.
