04 - CLI Basics

CLI commands are generally typed in this format: <command> <options> <arguments>. The options (or flags) usually start with -.
Example: ls -l /boot (ls is the utility or command, -l is the flag for "use a long listing format", and argument is the provided path /boot )

Help and Documentation

  • Some commands have built-in help “options”, usually -h, or -help, or --help
  • There are also Comments in config and example configuration files

man pages (The Manual)

Unix manuals were originally printed in multiple volumes and provided to customers. They were broken down into several sections. For example, User commands were in section 1.
In Linux and most Unix systems, the “man pages” are now software documentation included in the OS. This is the preferred way of looking up commands and documentation since the man pages provided with the system are often specific to that system.

The following are the default section numbers for man pages in most Linux and Unix systems:

Section Description
1 General Commands
2 System Calls
3 Library, functions, C standard library
4 Special Files (e.g device files and drivers)
5 File formats and conversions (e.g config files syntax)
6 Games and screensavers
7 Miscellany
8 System administration commands and daemons

The following are example commands to access the manpages for specific commands or configuration files:

user@hostmane:~$ man <command>
user@hostname:~$ man <section number> <command>
user@hostname:~$ whatis <command>

Linux Filesystem

In Linux, "everything is a file". All files are stored under root (/). Even other volumes such as attached USB drives and volumes in hard drives are "mounted" in any empty directory under /.

File Paths

There are two methods of addressing a specific file:

  • Absolute Path: starts from the root (/): /home/user/Documents/file1
  • Relative Path: Stars from your current location in the filesystem: ./Documents/file1

File Types in Linux

Unix/Linux have defined file types inside the filesystem. In fact, the Linux kernel does not look for file extensions to determine the file type. The file type can be easily viewed using ls -l command. The very first character on each line indicates the file type. For example lrwx------ indicates the filetype to be a "link".
Typical file types in a Unix/Linux filesystem are:

  • - : regular file
  • d : directory
  • c : character device file
  • b : block device file
  • s : local socket file
  • p : named pipe
  • l : symbolic link

For more information see: https://linuxconfig.org/identifying-file-types-in-linux.

  • Remember: file types in Linux are NOT images, documents, dll, etc…
  • A directory is just a special file that contains a list of files that are inside the directory

File Permissions

Every file has its permissions stored along its containing data inside the filesystem. You can see the permissions using the ls -l command.

  • r is for read, w is for write, and x is for execute
  • There are 3 sets of permissions on each file for the owner, group, and others

Here is a breakdown of what you get when using ls -l. Notice that the headers are added for your view:

In the above example, the file named acpi has the following information: drwxr-xr-x.

  • The first character d indicates that this is a directory.
  • The first set of 3 letter permissions rwx are for the owner of the file, and the owner is root
  • the second set of permissions r-x are for the group, which in this case the group name is also root. This indicates that members of the group root can read and execute the file, but cannot write to this file (cannot delete or modify).
  • The 3rd set, r-x, is for others, anyone who is not a member of the root group and is not the user, root.

Common Commands

  • List directory content: ls
  • Change Directory: cd
  • Display the present working directory: pwd
  • remove: rm
  • create a new empty file, or update timestamp on an existing file: touch
  • copy: cp
  • move: mv

Navigation using Shortcuts

  • ~ (tilde) refers to the user’s home directory. This refers to a variable called $HOME in bash. Example: cd ~
    • ~username to get to that user’s home directory, Example: cd ~user2
  • . (dot) refers to the current directory ("here" in the relative path!). Example: cd ./Documents
  • .. (dot dot) refers to one directory above. Example: cd ..

Working with Bash

Bash Tricks

  • Tab key to auto-complete command, path, or filename
  • Up and down arrow keys for command history
  • Special characters:
    • * (asterisk) Matches zero or more characters: ls *.txt
    • ? (question mark) Matches any single character: ls ????
    • [] (square brackets) Matches a single character in a range: ls [a][0-9]*
    • $ (dollar sign) Identifies a variable
    • ~ (tilde character) Represents the user home directory: cd ~/Documents
    • \ (backslash): turns off the special meaning of its following character: echo \$PATH outputs $PATH and will ignore the function of $
    • " (double quotes): cause most metacharacter special meanings to be ignored: echo "date"
    • ' (single quotes): negate the translation of all special characters: ls '?' outputs actual filename matching ?
    • ` (backquote): enable command substitution to occur. Exactly opposite of ‘single quotes’! it will take the input as a command: echo `date`outputs the date since it will execute date as a command

Redirection Commands

Redirection commands are used for redirecting input from or output to other sources such as a file, network, device, etc. (other than Standard input or keyboard).

  • redirection Symbols
    • < (take input from ... )
    • > (output to ...)
    • >> (output to and append to an existing file)
    • | (Pipe): takes the output of a command and connects it to the input of another command: ls -la | more
